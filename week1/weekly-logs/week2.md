# Week 2 — Git, GitHub & Version Control Workflows

## Topics Covered

- Git fundamentals (init, add, commit, branch)
- GitHub repository creation and management
- Conventional commits and clean commit history
- Branch management and pull request workflows
- Collaboration basics: forks and rebases (concepts)

## What I Practiced

- Creating and cloning a GitHub repository
- Writing a clear README with learning goals
- Making meaningful commits using Conventional Commits format
- Creating and switching branches
- Structuring weekly learning logs
- Using pull requests to merge changes (standard workflow)

## Key Takeaways

- Git tracks intent and history, not just files
- Clean commit messages improve readability and collaboration
- Branches help isolate work and reduce risk
- Pull requests make changes reviewable and traceable

## Challenges & Notes

- Understanding when to merge vs rebase (keeping history clean)
- Avoiding accidental commits using .gitignore
